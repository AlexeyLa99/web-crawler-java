# תרגיל 1 – תוכנית מימוש סופית (מבוססת על ex1.pdf)

> קובץ זה מסכם את כל הדרישות המאושרות מה-PDF הרשמי.

---

## מה אנחנו בונים?

אנחנו בונים **Web Crawler מקבילי** בשפת Java שעושה את הדברים הבאים:

- מתחיל מרשימת כתובות URL ב"קובץ זרעים" (input file)
- סורק דפים עד לעומק מקסימלי שניתן כפרמטר
- משתמש במספר threads דרך thread pool
- מחלץ מידע מכל דף
- מריץ ניתוחים (analyses) שהתבקשו
- כותב את התוצאות כ-JSON לקובץ פלט

---

## הרעיון המרכזי

המטלה **אינה** סדרתית (sequential).

במקום:
```
URL1 → URL2 → URL3
```

צריך שיהיו מספר threads שעובדים **במקביל** על דפים שונים בו-זמנית.

---

## זרימת הסריקה הבסיסית

1. קריאת כתובות ה-URL מקובץ הזרעים
2. הכנסתן לתהליך הסריקה עם עומק `0`
3. שליפת דף (fetch)
4. חילוץ הנתונים והקישורים היוצאים
5. אם העומק מאפשר – הוספת קישורים חדשים לתור
6. הימנעות מביקור חוזר באותה כתובת
7. המשך עד שהסריקה הושלמה

---

## דרישות מקביליות (Concurrency)

ה-PDF מדגיש במיוחד את איכות המקביליות:

- שימוש ב-thread pool
- שמירה על thread safety בכל המבנים המשותפים
- הימנעות מ-race conditions
- נעילה (locking) מינימלית בלבד
- אין busy-waiting
- אין חזרה למימוש סדרתי (זה פסול!)

**מבנים משותפים קריטיים שחייבים להיות thread-safe:**
- רשימת ה-URLs שכבר ביקרנו בהם
- רשימת תוצאות הדפים שנאספו

---

## ממשק שורת הפקודה (CLI)

```bash
java Main --analysis <strategies> --poolsize <n> --depth <d> --input <file> --output <file>
```

### פרמטרים חובה

| פרמטר | תיאור | ולידציה |
|-------|-------|---------|
| `--analysis` | רשימת ניתוחים מופרדת בפסיקים | לפחות אחד תקף, אחרת יציאה |
| `--poolsize` | גודל ה-thread pool | חייב להיות > 0 |
| `--depth` | עומק סריקה מקסימלי | חייב להיות >= 0 (0 = רק הדף הנוכחי) |
| `--input` | קובץ כתובות הזרעים | חייב להיות קריא |
| `--output` | נתיב קובץ הפלט (JSON) | נתיב לכתיבה |

### פרמטרים אופציונליים

| פרמטר | תיאור | ברירת מחדל |
|-------|-------|------------|
| `--format` | פורמט הפלט (`json` / `csv`) | `json` |
| `--domains` | רשימת דומיינים מותרים | ללא הגבלה |

---

## מה מחלצים מכל דף?

כל דף יוצג בפלט עם השדות הבאים:

| שדה | סוג | תיאור |
|-----|-----|-------|
| `url` | String | כתובת הדף |
| `title` | String | כותרת הדף (`<title>`) |
| `wordCount` | int | מספר המילים בטקסט הנראה |
| `outgoingLinks` | int | מספר הקישורים היוצאים |
| `domain` | String | שם הדומיין |
| `status` | int | קוד סטטוס HTTP |
| `depth` | int | עומק הגילוי |

---

## מבנה פלט ה-JSON

```json
{
  "pages": [
    {
      "url": "https://example.com",
      "title": "Example Domain",
      "wordCount": 42,
      "outgoingLinks": 3,
      "domain": "example.com",
      "status": 200,
      "depth": 0
    }
  ],
  "analysis": {
    "WORD_COUNT": 42,
    "MOST_LINKED_DOMAIN": "iana.org",
    "BROKEN_LINKS": ["https://example.com/missing"],
    "KEYWORD_FREQUENCY": {
      "java": 12,
      "thread": 5,
      "pattern": 3
    }
  }
}
```

**חשוב:**
- דפים עם סטטוס שאינו 200 **כן מופיעים** ב-`pages`, אבל `title`, `wordCount` ו-`outgoingLinks` יהיו null/0
- אם `BROKEN_LINKS` נבקש אך אין קישורים שבורים: `"BROKEN_LINKS": []`
- סדר הדפים = **סדר גילוי ראשון** (first-discovery order)

---

## הניתוחים (Analyses)

| שם | תיאור |
|----|-------|
| `WORD_COUNT` | סך כל המילים בכל הדפים |
| `MOST_LINKED_DOMAIN` | הדומיין עם הכי הרבה קישורים יוצאים אליו (שוויון – בחירה לקסיקוגרפית) |
| `BROKEN_LINKS` | רשימת URLs שהחזירו סטטוס שאינו 200 (בסדר גילוי) |
| `KEYWORD_FREQUENCY` | תדירות המילים: `java`, `thread`, `pattern` (לא תלוי רישיות) |
| `AVERAGE_WORD_COUNT` | **(הרחבה)** ממוצע מילים לדף (דפי HTTP 200 בלבד) |

---

## הודעות שגיאה (מדויקות!)

| מצב | הודעה | יעד | המשך? |
|-----|-------|-----|-------|
| URL לא תקין | `<url> malformed` | stderr | המשך |
| URL לא נגיש | `<url> failed` | stderr | המשך |
| כישלון בכתיבת קובץ פלט | `error saving report` | stdout | --- |
| כישלון בקריאת קובץ קלט | `invalid input file` | stdout | יציאה |
| poolsize <= 0 | `invalid pool size` | stdout | יציאה |
| depth < 0 | `invalid depth` | stdout | יציאה |
| שם ניתוח לא מוכר | `<name> is unknown` | stdout | מתעלם |
| אין ניתוח תקף | `no valid analysis` | stdout | יציאה |

---

## עיבוד דפים – כללים נוספים

- **HTTP 200 בלבד** מנותחים במלואם
- **הפניות (3xx)** עוקבים אחריהן אוטומטית עד 20 קפיצות (ברירת מחדל של Jsoup)
- **Timeouts:** 3 שניות לחיבור ו-3 שניות לקריאה
- **HTTP בלבד** – רק כתובות `http://` ו-`https://`

## Design Patterns – חובה לפחות 4

רשימת הדפוסים שמוזכרים ב-PDF:

| דפוס | שימוש שלנו |
|------|-----------|
| **Strategy** | `AnalysisStrategy` – החלפת אנליזות; `OutputWriter` – החלפת פורמט פלט |
| **Factory** | `AnalysisFactory`, `OutputWriterFactory` |
| **Builder** | `CrawlConfig.Builder` – בניית קונפיגורציה immutable |
| **Template Method** | `AbstractAnalysis` – שלד ניתוח, subclass ממלאים את הפרטים |
| **Observer** | `CrawlObserver` – עדכון בזמן אמת על דפים חדשים |
